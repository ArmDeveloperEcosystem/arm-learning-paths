Key Metrics to Analyze

  • TPS (Transactions Per Second): Higher is better.
  • QPS (Queries Per Second): Higher is better.
  • Latency (Average and 95th Percentile): Lower is better.

Conclusion

Merged BOLT profile for read+write performs similarly to the regular BOLT instrumentation. This indicates that the merge process was correctly executed and did not degrade performance. The experiment validates the applicability of BOLT profile merging for mixed workloads.

When you create a BOLT-instrumented binary using a read-only profile, BOLT reorders functions and basic blocks based entirely on what was hot during the read workload. Same for write only. But BOLT has to compromise layout to accommodate both read and write execution paths.
